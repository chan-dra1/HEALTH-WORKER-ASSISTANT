# HealthWorker

Offline-first health assistant for Community Health Workers (CHWs) in low-resource settings (Africa & India). Flutter mobile app, local SQLite storage, works with zero internet.

## Status

**v0.1 — Day 1 foundation.** Patient model + SQLite + home screen are in place. Symptom checker, dosage calculator, vitals tracker, and offline maps land on subsequent days per the 14-day roadmap.

## Features in this commit

- Patient CRUD backed by SQLite (`sqflite`)
- Home screen with patient list and add-patient dialog
- Data models: `Patient`, `Observation`, `Medication`
- Smoke tests for the `Patient` model

## Project layout

```
lib/
  main.dart                 # App entry point
  models/
    patient.dart            # FHIR-lite patient record
    observation.dart        # Vitals / measurements
    medication.dart         # Drug reference w/ weight-based dosing
  services/
    database_service.dart   # SQLite singleton + CRUD
  screens/
    home_screen.dart        # Patient list + add dialog
assets/
  data/                     # symptom_database.json, drug_reference.json (later)
  images/
test/
  patient_test.dart
pubspec.yaml
```

## Run it locally

The platform-specific folders (`android/`, `ios/`, etc.) are generated by Flutter and are **not** committed. After cloning, run:

```bash
flutter create .          # generates android/, ios/, etc. into this repo
flutter pub get
flutter run               # on a connected device or emulator
```

You can then add a patient via the `+` button and confirm the record persists across app restarts.
